Version 1.0.45 - Heise Developer Artikelserie Continuous Delivery Teil 2

Diese virtuelle Maschine (VM) ist die "Begleit VM" zum zweiten Artikel "Die Commitstage" der auf Heise Developer erscheinenden Artikelserie zu Continuous Delivery.

Auf der VM sind alle für den exemplarischen Aufbau einer Continuous Delivery Pipeline notwendigen Komponenten vorinstalliert. Der Artikel "Die Commitstage" enthält eine Schritt-für-Schritt Anleitung, die Sie mit Hilfe dieser VM leicht nachvollziehen können.

Der Artikel nimmt dabei Bezug auf diese vorkonfigurierten Komponenten:

MusicDB - das Java JEE Beispielprojekt

Der Sourcecode des Beispielprojekts "MusicDB" liegt ausgecheckt im Eclipse Workspace des angemeldeten Benutzers. Wenn Sie die IDE (Klick auf Icon ) starten, können Sie den Sourcecode editieren, die Unittests in der IDE laufen lassen, Änderungen ins "zentrale" Git Repository pushen und die Anwendung selbst in einen lokalen Tomcat deployen. Die Anwendung selbst erreichen Sie dann unter MusicDB

Zentrales Versionskontrollsystem

Continuous Delivery Pipelines verwenden immer (mindestens) ein zentrales Sourcecode Repository. Wir haben ein einfaches Git Repository in das Dateisystem gelegt: /srv/git-repo. Mit Gitweb können Sie komfortabel durch die Inhalte dieses Repositories browsen.

Git Repo »

CI-Server Jenkins

Unter Jenkins finden Sie eine vorkonfigurierte Instanz des CI-Servers Jenkins. Alle nötigen Plugins sind installiert. Sie können also sofort die im Artikel beschriebenen Schritte zur Erstellung der diversen Jobs nachvollziehen.

Jenkins »

Bundle Repository

Alle von der Pipeline erzeugten Binär-Artefakte werden in einem einfachen lokalen Repository abgelegt. Es handelt sich um eine simple Orderstruktur im Dateisystem unter /data/repo. Als einfache WebUI zum Repository dient ein Apache Webserver.
